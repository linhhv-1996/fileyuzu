## 바코드는 어떤 상황에서 필요할까요?

바코드는 단순히 검은색 선이 모여 있는 이미지처럼 보이지만, 핵심은 “정보를 기계가 빠르게 읽을 수 있게 만든 코드”입니다. 사람이 긴 숫자나 문자를 직접 입력하지 않아도, 스캐너나 카메라가 바코드를 읽어 해당 값을 빠르게 인식할 수 있습니다.

일상적인 사용 예시는 다음과 같습니다.

- 상품 코드 관리
- 재고 번호 관리
- 창고 위치 표시
- 사무실 자산 라벨
- 도서 또는 문서 관리
- 이벤트 티켓 번호
- 쿠폰 코드
- 회원 번호
- 내부 생산 번호
- 배송 또는 포장 구분

특히 작은 쇼핑몰이나 개인 사업자는 처음부터 복잡한 재고 관리 시스템을 도입하기 어렵습니다. 이럴 때 상품마다 내부 코드를 만들고, 그 코드를 바코드 이미지로 출력해 붙여두면 관리가 훨씬 쉬워집니다.

예를 들어 상품 코드가 `ITEM-0001`, `ITEM-0002`, `ITEM-0003`처럼 정리되어 있다면, 해당 코드를 바코드로 만들어 라벨에 붙일 수 있습니다. 이후 상품을 확인할 때 코드를 직접 입력하지 않고 스캔으로 처리할 수 있습니다.

물론 모든 바코드가 공식 유통용 바코드는 아닙니다. 마트, 편의점, 대형 유통망에서 사용하는 공식 상품 바코드는 별도의 코드 체계와 등록 절차가 필요할 수 있습니다. 하지만 내부 관리용, 창고용, 테스트용, 문서용, 라벨용 바코드는 직접 만들어도 충분히 유용합니다.

## 바코드 만들기 전에 준비해야 할 것

바코드를 만들기 전에 먼저 정해야 할 것이 있습니다. 아무 코드나 입력해도 이미지 자체는 만들 수 있지만, 실제로 사용할 목적이 있다면 조금 더 신중하게 준비하는 것이 좋습니다.

### 1. 바코드에 넣을 코드

가장 먼저 필요한 것은 바코드에 들어갈 값입니다. 이 값은 숫자일 수도 있고, 문자와 숫자가 섞인 코드일 수도 있습니다.

예를 들면 다음과 같습니다.

- `8801234567890`
- `ITEM-0001`
- `SKU-A102`
- `BOOK-2026-001`
- `ASSET-00045`

중요한 것은 나중에 관리하기 쉬운 규칙을 만드는 것입니다. 상품마다 임의로 코드를 만들면 처음에는 괜찮아 보여도, 상품이 늘어나면 중복이나 혼란이 생길 수 있습니다.

예를 들어 의류 쇼핑몰이라면 카테고리, 색상, 사이즈를 코드에 반영할 수도 있습니다.

```text
TSHIRT-BLK-M-001
TSHIRT-WHT-L-002
PANTS-BLU-XL-003
````

이런 식으로 규칙을 정해두면 바코드 자체뿐 아니라 상품 관리도 쉬워집니다.

### 2. 바코드 아래에 표시할 라벨

바코드 이미지만 있으면 사람이 보기에는 어떤 상품인지 바로 알기 어렵습니다. 그래서 바코드 아래에 코드 값이나 상품명을 함께 표시하는 경우가 많습니다.

예를 들어 바코드 아래에 `ITEM-0001` 또는 `상품 A`처럼 라벨을 넣으면 인쇄 후에도 사람이 쉽게 확인할 수 있습니다.

UploadLess 바코드 생성기에서는 코드와 라벨을 함께 사용할 수 있습니다. 특히 여러 바코드를 만들 때는 label 값을 잘 정리해두면 출력 후 확인이 훨씬 편합니다.

### 3. 사용할 바코드 형식

바코드에는 여러 종류가 있습니다. 일반적인 상품 코드에는 EAN-13이나 UPC-A가 사용되는 경우가 많고, 내부 관리용 코드에는 Code 128 같은 형식이 자주 사용됩니다.

처음 바코드를 만드는 사람이라면 복잡하게 생각할 필요는 없습니다. 내부 관리용으로 문자와 숫자가 섞인 코드를 만들고 싶다면 Code 128이 비교적 유연합니다. 숫자만으로 된 상품 코드 형태를 만들고 싶다면 EAN/JAN 계열을 고려할 수 있습니다.

다만 공식 유통망에서 사용하는 상품 바코드는 단순히 이미지를 만드는 것과는 다릅니다. 실제 판매 채널이나 유통사에서 요구하는 코드 체계를 확인해야 합니다.

### 4. 저장할 이미지 형식

바코드를 만들고 나면 이미지로 저장해야 합니다. 이때 많이 사용하는 형식은 PNG와 SVG입니다.

PNG는 일반 이미지 파일입니다. 문서, 블로그, 쇼핑몰 상세페이지, 간단한 라벨 작업에 넣기 쉽습니다.

SVG는 벡터 이미지입니다. 크기를 키워도 선명도가 유지되기 때문에 인쇄용이나 디자인 편집용으로 좋습니다. 바코드를 크게 출력하거나 라벨지에 넣어 인쇄할 계획이라면 SVG가 유리할 수 있습니다.

## PNG와 SVG 중 어떤 형식으로 다운로드해야 할까요?

바코드를 만들고 나면 어떤 이미지 형식으로 저장해야 할지 고민할 수 있습니다. 일반적으로는 PNG와 SVG를 많이 사용합니다.

### PNG가 좋은 경우

PNG는 가장 익숙한 이미지 형식 중 하나입니다. 대부분의 문서 편집기, 이미지 편집기, 웹사이트, 쇼핑몰 관리자 페이지에서 쉽게 사용할 수 있습니다.

다음과 같은 상황에서는 PNG가 편합니다.

* 문서에 바코드 이미지를 넣을 때
* 블로그나 웹페이지에 바코드를 넣을 때
* 쇼핑몰 상세페이지에 샘플 이미지를 넣을 때
* 간단한 라벨 작업에 사용할 때
* 다른 사람에게 이미지 파일로 전달할 때

PNG는 사용하기 쉽다는 장점이 있습니다. 파일을 다운로드한 뒤 바로 열어볼 수 있고, 대부분의 프로그램에서 문제 없이 삽입할 수 있습니다.

### SVG가 좋은 경우

SVG는 벡터 형식입니다. 일반 이미지처럼 픽셀로 고정되어 있는 것이 아니라 선과 도형 정보로 구성됩니다. 그래서 크기를 키워도 선명하게 유지됩니다.

다음과 같은 상황에서는 SVG가 좋습니다.

* 인쇄용 라벨을 만들 때
* 바코드를 크게 출력해야 할 때
* 디자인 툴에서 편집해야 할 때
* 선명한 품질이 중요한 경우
* 여러 크기로 재사용해야 할 때

바코드는 선명도가 중요합니다. 인쇄 후 선이 흐려지거나 너무 작게 출력되면 스캐너가 제대로 읽지 못할 수 있습니다. 그래서 인쇄용으로 사용할 계획이라면 SVG를 선택하는 것이 안전한 경우가 많습니다.

## 상품 바코드 만들 때 주의할 점

상품 바코드를 만들 때 가장 많이 헷갈리는 부분은 “내가 만든 바코드를 실제 판매 상품에 바로 붙여도 되는가?”입니다.

답은 상황에 따라 다릅니다.

내부 관리용이라면 직접 만든 코드와 바코드를 사용할 수 있습니다. 예를 들어 작은 창고에서 상품을 구분하거나, 사무실 자산을 관리하거나, 온라인 판매 전에 내부적으로 재고를 정리하는 용도라면 직접 만든 코드도 충분히 활용할 수 있습니다.

하지만 오프라인 매장, 대형 유통사, 마켓플레이스, 국제 유통망에서 공식적으로 사용하는 상품 바코드는 별도의 코드 등록이 필요할 수 있습니다. 이런 경우에는 단순히 바코드 이미지를 만드는 것만으로는 충분하지 않습니다. 해당 유통 채널이 요구하는 코드 체계를 확인해야 합니다.

즉, 바코드 이미지를 만드는 것과 공식 상품 코드를 발급받는 것은 다른 문제입니다.

정리하면 다음과 같습니다.

* 내부 재고 관리용: 직접 생성 가능
* 사무실 자산 관리용: 직접 생성 가능
* 테스트용/샘플용: 직접 생성 가능
* 공식 유통 상품용: 별도 코드 체계 확인 필요
* 대형 판매 채널 등록용: 해당 플랫폼 요구사항 확인 필요

이 차이를 이해하고 사용하면 바코드를 더 안전하게 활용할 수 있습니다.

### 충분한 여백을 확보하세요

바코드 주변에는 여백이 필요합니다. 바코드 선 바로 옆에 글자나 이미지가 붙어 있으면 스캐너가 바코드 영역을 정확히 인식하지 못할 수 있습니다.

라벨 디자인을 할 때는 바코드 주변에 충분한 빈 공간을 남겨두는 것이 좋습니다.

### 너무 작게 출력하지 마세요

디자인을 위해 색상을 바꾸고 싶더라도 실제 스캔이 되는지 반드시 확인해야 합니다.

### 라벨 텍스트를 확인하세요

바코드 아래에 표시되는 라벨이 실제 코드와 일치하는지 확인해야 합니다. 사람이 읽는 텍스트와 바코드 데이터가 다르면 관리 과정에서 혼란이 생길 수 있습니다.

### 출력 후 테스트 스캔을 하세요

가장 중요한 것은 실제 테스트입니다. 바코드 이미지를 만들고 라벨에 넣고 출력한 다음, 실제 사용하는 스캐너나 스마트폰으로 읽어보세요. 화면에서는 정상이어도 인쇄 품질에 따라 결과가 달라질 수 있습니다.

## UploadLess 바코드 생성기의 3가지 모드

[UploadLess 바코드 생성기](/ko/barcode-generator)는 단순히 하나의 바코드만 만드는 도구가 아니라, 상황에 맞게 세 가지 방식으로 바코드를 만들 수 있습니다.

### 1. 수동 모드

수동 모드는 코드를 직접 입력해서 바코드를 만드는 방식입니다. 바코드를 몇 개만 빠르게 만들고 싶을 때 좋습니다.

예를 들어 다음과 같은 상황에 적합합니다.

* 테스트용 바코드 만들기
* 상품 코드 하나만 바코드로 변환하기
* 문서에 넣을 샘플 바코드 만들기
* 라벨 디자인 확인용 바코드 만들기

가장 단순하고 빠른 방식이기 때문에 처음 사용하는 사람에게도 편합니다.

### 2. 파일 업로드 모드

파일 업로드 모드는 CSV 또는 Excel 파일을 사용해 여러 개의 바코드를 한 번에 만드는 방식입니다.

파일은 기본적으로 다음과 같이 준비하면 됩니다.

| code          | label |
| ------------- | ----- |
| 8801234567890 | 상품 A  |
| 8801234567891 | 상품 B  |
| 8801234567892 | 상품 C  |

A열에는 바코드에 들어갈 `code`, B열에는 바코드 아래에 표시할 `label`을 넣으면 됩니다.

상품 목록이나 재고 목록이 이미 Excel로 정리되어 있다면 이 방식이 훨씬 빠릅니다. 하나씩 복사해서 입력할 필요 없이 파일을 업로드하고 여러 바코드를 한 번에 만들 수 있습니다.

### 3. 순차 번호 모드

순차 번호 모드는 연속된 번호로 바코드를 만들 때 유용합니다.

예를 들어 다음과 같은 코드가 필요할 수 있습니다.

```text
ITEM-0001
ITEM-0002
ITEM-0003
ITEM-0004
ITEM-0005
```

이런 번호를 하나씩 입력하는 것은 귀찮고 실수도 생기기 쉽습니다. 순차 번호 모드를 사용하면 시작 번호와 개수를 기준으로 연속된 바코드를 빠르게 만들 수 있습니다.

자산 관리, 티켓 번호, 쿠폰 번호, 재고 번호처럼 순서가 있는 코드를 만들 때 특히 유용합니다.

## 바코드 생성 도구를 선택할 때 확인할 점

온라인 바코드 생성 도구는 많지만, 실제로 사용해보면 차이가 있습니다. 단순히 바코드 이미지를 보여주는 것만으로는 부족할 수 있습니다.

좋은 바코드 생성 도구라면 다음 요소를 확인하는 것이 좋습니다.

### 다양한 입력 방식을 지원하는가?

하나씩 입력하는 방식만 있으면 바코드가 많아질 때 불편합니다. 수동 입력뿐 아니라 파일 업로드, 순차 번호 생성 같은 방식이 있으면 훨씬 실용적입니다.

### PNG와 SVG 다운로드를 지원하는가?

웹이나 문서용으로는 PNG가 편하고, 인쇄용으로는 SVG가 좋습니다. 둘 다 지원하는 도구를 선택하면 상황에 맞게 사용할 수 있습니다.

### 라벨 표시를 지원하는가?

바코드 아래에 코드나 상품명을 표시할 수 있으면 실제 라벨 작업이 쉬워집니다. 특히 여러 상품 바코드를 출력할 때는 라벨이 큰 도움이 됩니다.

### 사용 방법이 간단한가?

도구가 아무리 기능이 많아도 사용하기 복잡하면 오래 쓰기 어렵습니다. 파일을 넣고, 옵션을 고르고, 바로 다운로드할 수 있는 흐름이 중요합니다.

### 무료로 사용할 수 있는가?

간단한 바코드 생성 작업이라면 비싼 프로그램이 필요하지 않습니다. 무료 온라인 도구로 충분한 경우가 많습니다.
