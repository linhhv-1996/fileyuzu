## 한글 웹 환경에서 PDF 용량 축소가 유독 까다롭고 번거로웠던 이유

단순히 영문 알파벳과 선형 텍스트 위주로 구성된 서구권의 서류들과 달리, 우리가 매일 다루는 국내 비즈니스 문서 및 프리젠테이션 자료들은 다음과 같은 독특한 디자인적, 기술적 특성 때문에 일반적인 범용 압축 엔진을 돌리면 파일이 깨지거나 심각한 품질 저하가 발생하기 쉽습니다.

### 1. 고해상도 이미지 스캔 본의 높은 비중
국내 오피스 환경에서는 종이로 된 계약서나 영수증, 도면 서류를 복합기로 스캔하여 PDF로 변환하는 경우가 매우 잦습니다. 이러한 스캔 문서들은 내부 구조상 텍스트 레이어가 아니라 하나의 거대한 고해상도 통이미지로 저장되기 때문에 내부 픽셀 밀도가 무지막지하게 높습니다. 기술력이 부족한 부실한 엔진으로 무작정 압축을 진행하면, 전체 용량은 줄어들지 몰라도 가장 중요한 글씨 구석구석에 자글자글한 디지털 노이즈가 발생하여 가독성이 처참하게 망가지는 현상이 일어납니다.

### 2. 복잡한 조합형 폰트와 메타데이터의 팽창
한글 문서는 초성, 중성, 종성이 결합하는 특성상 폰트 파일 자체의 용량이 영어 대비 수십 배 무겁습니다. 문서를 PDF로 저장할 때 가독성을 위해 폰트 데이터를 문서 내부에 완전히 포함(Embedded)시키는 경우가 많은데, 이 과정에서 눈에 보이지 않는 중복 메타데이터와 레이어 찌꺼기들이 결합하여 파일 크기를 불필요하게 키우게 됩니다. 

이러한 문제를 깔끔하게 극복하기 위해서는 이미지의 화질을 강제로 떨어뜨리는 원시적인 방식이 아니라, 문서 내부의 폰트 구조와 중복 데이터를 분석하여 덜어내는 고도화된 최적화 기술이 필요합니다.

**Q4. 파일 업로드 중에 인터넷 연결이 끊기면 어떻게 되나요?**
기존 서버 기반 서비스들과 달리, 파일이 외부로 전송되는 과정이 없으므로 사이트에 접속만 한 상태라면 도중에 인터넷(와이파이) 연결이 끊어져도 압축 프로세스는 100% 완벽하게 정상 진행되고 결과물을 다운로드할 수 있습니다. 

**Q5. 맥북(Mac)이나 아이폰 환경에서도 정상 구동되나요?**
