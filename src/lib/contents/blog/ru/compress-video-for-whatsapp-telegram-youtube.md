## Ограничения по размеру на популярных платформах

Прежде чем сжимать видео, полезно понимать, с какими ограничениями вы работаете.

| Платформа | Лимит на файл | Рекомендуемый формат |
|-----------|--------------|----------------------|
| WhatsApp | 16 МБ (мобайл) / 64 МБ (десктоп) | MP4, H.264 |
| Telegram | 2 ГБ (бот) / 4 ГБ (премиум) | MP4 |
| Email (Gmail, Outlook) | 25 МБ | MP4 |
| Instagram (Reels) | 1 ГБ | MP4, MOV |
| TikTok | 287 МБ (iOS) / 72 МБ (Android) | MP4, MOV |
| YouTube | 256 ГБ / 12 часов | MP4, MOV, AVI |
| Facebook | 4 ГБ / 240 минут | MP4, MOV |

Большинство ограничений существуют именно на уровне файла, а не качества. Это означает, что при правильном сжатии вы получаете визуально хорошее видео, которое без проблем проходит через лимиты.

## Сжать видео для WhatsApp

WhatsApp — самый жёсткий ограничитель из мессенджеров. На мобильных устройствах лимит составляет всего 16 МБ, на десктоп-версии — 64 МБ. Стандартная минута видео в 1080p с телефона легко превышает 200 МБ.

Для отправки через WhatsApp используйте режим **Целевой размер файла**: задайте 15 МБ (с небольшим запасом до лимита) и нажмите кнопку. Инструмент рассчитает оптимальный битрейт для этого размера. Видео длиной до 2–3 минут обычно сохраняет вполне приемлемое качество при таком уменьшении.

## Сжать видео для Telegram

Telegram значительно лояльнее: лимит составляет 2 ГБ для обычных пользователей и 4 ГБ для Telegram Premium. Но даже здесь сжатие может быть полезным — меньший файл загружается быстрее и не занимает место у получателя.

Для Telegram лучше подойдёт режим **Автооптимизация**: он найдёт баланс между качеством и размером без необходимости задавать конкретную цифру. Если нужно уложиться в определённый размер — например, для отправки через бота с ограничением 50 МБ — используйте режим целевого размера.

## Сжать видео для YouTube

YouTube не имеет жёстких ограничений по размеру (до 256 ГБ), но большие файлы долго загружаются. При слабом соединении загрузка 4K-видео в 10 ГБ может занять несколько часов.

Для YouTube имеет смысл применить **Автооптимизацию**: она снизит размер файла, сохранив при этом высокое разрешение. Важно: YouTube всё равно перекодирует загруженное видео на своей стороне, поэтому отправлять исходный файл с максимальным битрейтом необязательно — разницы в итоговом качестве на платформе практически нет.

## Сжать видео для Instagram

Instagram автоматически пережимает видео при загрузке — и делает это достаточно агрессивно. Если вы загружаете очень большой файл, платформа применяет дополнительное сжатие, которое заметно ухудшает картинку.

Парадокс: заранее сжатый файл умеренного размера часто выглядит на Instagram лучше, чем оригинал. Алгоритм Instagram «видит», что файл уже оптимизирован, и применяет меньше дополнительного сжатия.

Для Instagram используйте **Автооптимизацию**. Если вы публикуете Reels в вертикальном формате 9:16, убедитесь, что разрешение сохранено — инструмент не меняет разрешение, только битрейт.

## Сжать видео для TikTok

У TikTok жёсткие ограничения на Android: всего 72 МБ. На iOS — до 287 МБ. Видео в 1080p длиной 60 секунд легко превышает оба лимита.

Для TikTok выберите режим **Целевой размер**: 70 МБ для Android или 280 МБ для iOS. Ролики в TikTok редко длиннее 3 минут, поэтому при умеренном сжатии качество картинки обычно остаётся хорошим.

## Сжать видео для Email

Email — самый строгий лимит: большинство почтовых сервисов (Gmail, Outlook, Mail.ru) ограничивают вложения до 25 МБ. Это практически исключает прямую отправку видео без предварительного сжатия.

Режим **Целевой размер** здесь незаменим: укажите 22–23 МБ (с запасом на другие вложения в письме), и инструмент рассчитает параметры. Видео длиной 1–2 минуты при таком размере обычно выглядит вполне приемлемо.

## Часто задаваемые вопросы

**Меняется ли разрешение видео при сжатии?**
Нет. Инструмент уменьшает битрейт, но сохраняет исходное разрешение. Ваш 1080p-файл останется 1080p — просто займёт меньше места.

**Что лучше использовать для WhatsApp — автооптимизацию или целевой размер?**
Для WhatsApp рекомендуем режим целевого размера: задайте 15 МБ для мобильной отправки или 60 МБ для десктопа, чтобы точно уложиться в лимит.

**Видео обрабатывается на сервере?**
Нет. Весь процесс происходит локально в вашем браузере с помощью WebAssembly. Файл не покидает ваше устройство.

**Можно ли сжать несколько видео?**
Сейчас инструмент обрабатывает файлы по одному. Для пакетной обработки запустите несколько вкладок браузера.

**Сохраняется ли звук после сжатия?**
Да, аудиодорожка сохраняется полностью.
